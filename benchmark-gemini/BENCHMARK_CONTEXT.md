# Contexto do benchmark

Você está participando de um benchmark comparativo entre ferramentas de IA para desenvolvimento frontend.

O objetivo é criar uma aplicação de calculadora genérica funcional usando React, TypeScript e Vite.

## Restrições

- Não use backend.
- Não use bibliotecas de UI prontas, como Material UI, Chakra, Ant Design ou Bootstrap.
- Pode usar CSS puro, CSS Modules ou outra abordagem simples.
- O projeto deve ser fácil de instalar, rodar e testar.
- Priorize código claro, modular, testável e acessível.
- Não adicione funcionalidades fora do escopo.
- Não ignore erros de build, lint ou testes.
- Ao final, explique o que foi implementado e quais comandos foram usados para validar.

## Critérios de qualidade

A entrega será avaliada por:

- Aderência ao escopo.
- Tempo de entrega.
- Quantidade de intervenção manual.
- Qualidade funcional.
- Qualidade visual.
- Qualidade do código.
- Testabilidade.
- Acessibilidade básica.
- Ausência de bugs críticos.
